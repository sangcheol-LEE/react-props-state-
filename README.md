## 각각의 커밋과 함께 코드를 보면 이해하는데 도움이 되실겁니다.

## Props

props는 읽기 전용의 속성

부모에서 자식에게 전해주는 속성이고 읽을수만 있다.

부모컴포넌트인 Parents.js 에서 자식컴포넌트인 children.js 쪽으로
property인 name,age,introduce를 전달 해주는 코드이고,
첫번째 컴포넌트에는 introduce라는 props가 있고,
두번째 컴포넌트에는 없습니다.

그래서 첫번째 컴포넌트는 버튼은 클릭하면 (onClick시 introduce함수가 호출) 되면서 경고창이 나오지만, 두번째 컴포넌트는 props로 전달해주지 않았기 때문에 버튼을 클릭해도 함수는 동작하지 않습니다.

## State

state는 상태라는 뜻

컴포넌트 자체에서 가지고 있는 값이고 내가 원한다면 값을 변경할 수 있다.
state를 만들 땐
constructor 라는 메소드를 사용해야한다.
그 이후 super () 메소드 사용

constructor() {
super ()
this.state={};
}

이런 형식으로 작성해준다. 클래스형 컴포넌트에서만 !

this 라는 건 내가 속해있는 컴포넌트를 지칭하는 단어 ! (간략하게 표현 시)

console을 render안에서 찍는 이유는 ?
=> 리액트는 화면을 그릴 때 render라는 함수를 호출해서 화면을 띄운다.

## state 변경하기 (이벤트를 통해서 변경하기.)

이벤트 리스너를 등록하는 식으로 변경하기

먼저 이벤트리스너에 들어갈 콜백함수 생성

- 컴포넌트 안에서 함수를 생성할 시 무조건 화살표 함수로 작성해야함

state 값은 직접 변경하면 안됩니다.
랜더 메소드가 다시 호출이 안됩니다.
고로 화면에는 변한 값이 출력되지 않습니다.

그렇기 때문에 setState() 라는 메소드를 사용해서 state 값을 변경시켜줍니다.
setState를 사용해야 지금 화면 다시 그려야한다고 말해주기 위해서 입니다.
이걸 사용해야 리액트가 화면을 바꿔야하는지 아닌지 알 수 있습니다.

## Props 와 State의 차이

props는 부모에서 자식으로 넘겨주는 것이고, state는 스스로가 가지고 있는 것.
받는 props에 값에 따라서 결과를 다르게 출력해야 할 때 사용하고 읽기 전용의 값.

state는 setstate를 사용해서 값을 언제든지 변경 할 수 있다.

## 리펙토링
